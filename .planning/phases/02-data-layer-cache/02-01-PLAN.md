<threat_model>
db.py change adds no runtime surface (no endpoints/auth/input/network/crypto). Threats:
- **T-2-01 Durability gap** — current `_atomic_write` has no fsync (`db.py:104-114`); a crash after `os.replace` can truncate. Mitigation: fsync temp file + parent dir (POSIX-guarded). Low severity, real.
- **Read-modify-write race** under the auto-refresh worker thread vs event-loop reads (the one concurrency point in `serve`). Mitigation: per-entry RLock; reads return immutable snapshots. Correctness, not security.
No secrets; cache is process-local.
</threat_model>
