# FC 26 Strategy Advisor (`fc26 advise`) — Design Spec

**Phase:** 10. **Date:** 2026-06-13. **Branch:** claude/fc26-strategies-lineups-umnhqc

## Goal

A read-only `fc26 advise <squad.json>` that turns a squad into **actionable strategy tips**, computed entirely from existing primitives (no new model):

1. **Chem leverage** — club/league/nation groups closest to their next chemistry tier. `ChemReport.tiers[].next_tier_at` already exposes the threshold; `needed = next_tier_at - count`. Surfaces "+1 more Spain nation → tier up (lifts every Spanish player's chem)". Sorted fewest-needed first.
2. **Out of position** — players whose slot position ≠ their card position (`SlotChem.in_position == False`, 0 chem). Reposition or replace.
3. **Weakest slots** — lowest `meta_score` players (bottom 3): the upgrade targets `plan`/`upgrade` should hit first.
4. **Best chem style** — per outfield slot, the style (of `available_styles()`) that most raises `meta_score` at that slot's current chem level, via `boosted_stats`. GK and 0-chem slots get an explanatory note, not a pick.

`advise` only diagnoses; `plan`/`upgrade` act. No `--budget`, no `--write` (YAGNI).

## Reuse (no changes to existing modules)

`compute_chemistry` (tiers + per-slot chem + in_position), `meta_score`, `boosted_stats` + `available_styles`, `slot_position`, `load_lineup`/`resolve_cards`, CLI `_fail`/`DB_OPTION`/`JSON_FLAG`. Best-style uses `dataclasses.replace(card, face=boosted.face)` then `meta_score` — argmax over styles.

## New module: `fc26/builder/advise.py`

```
@dataclass(frozen=True) TierLeverage: kind, name, count:int, points:int, needed:int
@dataclass(frozen=True) SlotNote:     slot, player_name, position, meta: float|None, chem:int, reason:str
@dataclass(frozen=True) StyleAdvice:  slot, player_name, chem:int, recommended_style: str|None, meta_gain: float, note: str|None
@dataclass(frozen=True) Advice:
  formation:str, team_chem:int,
  tier_leverage: tuple[TierLeverage,...],   # next_tier_at set, sorted by needed
  out_of_position: tuple[SlotNote,...],
  weakest_slots: tuple[SlotNote,...],       # bottom WEAKEST_SLOTS(=3) by meta
  style_advice: tuple[StyleAdvice,...],
  summary: tuple[str,...], warnings: tuple[str,...]

advise_squad(lineup, slot_cards) -> Advice     # named _squad to avoid the CLI `advise` clash
```

Style branch per slot: GK → `(None, "GK - style advice n/a")`; missing face meta → `(None, "missing face stats")`; chem ≤ 0 → `(None, "0 chem - style inert")`; else argmax style + `meta_gain` (or `(None, "no style raises pace-meta")`).

## CLI: `fc26 advise <squad.json> [--db] [--json]`

Mirrors `chem`: `load_lineup` → `resolve_cards` → `advise_squad`; `FC26Error` → `_fail`. Non-JSON: a summary bullet list, then sections (Out of position / Chem leverage table / Weakest slots table / Best chem styles table) + `warn:` lines. JSON: `asdict(advice)`.

## Acceptance

- Unit (`tests/test_advise.py`): structure; out-of-position flagged; tier leverage non-empty + ascending by `needed` + `needed>0`; weakest = lowest meta; best style is a real style with `meta_gain>0` and equals the recomputed argmax; 0-chem/GK/missing-face notes; JSON-serializable.
- CLI (`tests/test_cli.py`): table + json (`mode`-free Advice); clean lineup/missing-card errors (no `Traceback`); real-DB CI-guard.
- ≥80% coverage maintained; existing tests stay green.
