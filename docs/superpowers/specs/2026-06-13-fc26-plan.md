# FC 26 Acquisition Planner (`fc26 plan`) — Design Spec

**Phase:** 9 (post-frontend). **Date:** 2026-06-13. **Branch:** claude/fc26-strategies-lineups-umnhqc

## Goal

A new `fc26 plan` command that turns a budget into an **ordered, budget-phased acquisition ledger**: what to buy *now*, then the ordered upgrade path with running spend, remaining budget, a score/chem trajectory, and return-on-investment (squad-score gained per coin) for each buy.

## Why it's distinct from `build` / `upgrade`

| Command | Answers | Output |
|---|---|---|
| `build` | "best XI for budget B?" | final XI + one total cost |
| `upgrade` | "improve this squad within B?" | unordered-feeling swap table |
| **`plan`** | "**in what order** do I spend, and **what does each buy return**?" | step-by-step ledger: cumulative spend, remaining, score/chem after each step, ROI per coin |

The new value is **sequencing + cash-flow + ROI presentation**, not a new optimizer. `plan` reuses the existing greedy engines (`build_squad`, `find_upgrades`) as-is. A globally-optimal knapsack solver is explicitly **out of scope** (YAGNI; parked follow-up).

## Two modes (mirrors `build` / `upgrade` arg shapes)

- **Build mode** — `fc26 plan --formation 4-2-3-1 --budget 500K [--league L]`: run `build_squad`; present the cheapest legal **seed XI to buy now** + the ordered upgrade path.
- **Upgrade mode** — `fc26 plan squad.json --budget 200K [--swaps N]`: run `find_upgrades`; present the swap path as a running buy/sell ledger (no seed block).

Mode is chosen by presence of the positional `squad_file`. `--formation` with a squad file, or neither, is a clean `_fail`.

## Reuse (no changes to existing modules)

`plan.py` composes entirely from **public** outputs:
- `build_squad(...) -> BuildResult{lineup, slot_cards(final), seed_cost, improve_plan: UpgradePlan, total_cost}`
- `find_upgrades(...) -> UpgradePlan{swaps, spent, budget, score_before, score_after, chem_before, chem_after, warnings}`
- `Swap{slot, out_name, out_version, out_resale, in_name, in_version, in_price, net_cost, meta_delta, chem_delta, score_delta, in_id, out_id}`
- Baseline score/chem = `UpgradePlan.score_before / chem_before` (for build mode this is the seed squad). Trajectory = running sum of per-`Swap` deltas.
- Seed cards reconstructed from final `slot_cards` by reverting each swap (`seed[swap.slot] = pool_by_id[swap.out_id]`).
- `parse_budget`, `CardRepository`, `compute_chemistry`, `load_lineup`, `resolve_cards`, `_fail`, `DB_OPTION`, `JSON_FLAG` — all existing.

## New module: `fc26/builder/plan.py`

```
@dataclass(frozen=True) SeedBuy: slot, card_id, player_name, version, price:int
@dataclass(frozen=True) PlanStep:
  index:int, slot, out_name, out_version, out_resale:int,
  in_name, in_version, in_price:int, net_cost:int,
  cumulative_spent:int, remaining:int,
  score_after:float, chem_after:int, roi: float | None   # score_delta/net_cost; None when net_cost<=0 (free/profit)
@dataclass(frozen=True) AcquisitionPlan:
  mode:str("build"|"upgrade"), formation:str,
  seed: tuple[SeedBuy,...], seed_cost:int, base_score:float, base_chem:int,
  steps: tuple[PlanStep,...], budget:int, upgrade_spent:int, total_spent:int,
  final_score:float, final_chem:int, warnings: tuple[str,...]

plan_from_scratch(formation, pool, *, budget, league=None, name=None) -> AcquisitionPlan   # mode="build"
plan_for_squad(lineup, slot_cards, pool, *, budget, max_swaps=DEFAULT_MAX_SWAPS) -> AcquisitionPlan  # mode="upgrade"
```

`roi` is `None` (not `inf`) when `net_cost <= 0` to keep `--json` standards-valid.

## CLI: `fc26 plan`

Options: positional `squad_file: Path = Argument(None)`, `--formation`, `--budget` (required), `--league`, `--swaps` (default 3, upgrade mode), `--write`, `--db`, `--json`. Follows `build`/`upgrade` exactly: `parse_budget` → engine → render; `FC26Error` → `_fail`; `--write` saves final squad with the NEW-file guard; warnings as `[yellow]warn:[/yellow]`.

Table (build mode): a **seed** table (Slot/Player/Version/Price + `seed_cost`), then a **path** table (Step/Slot/Out→In/Net/Cumulative/Remaining/Score/Chem/ROI). Summary: `total cost {total_spent} of {budget} | squad score {base_score:.1f} → {final_score:.1f} | chem {base_chem} → {final_chem}`. Upgrade mode: path table only. Empty path → `"no upgrades found within budget"` (build mode still prints the seed).

## Acceptance

- Unit (`tests/test_plan.py`): build & upgrade plans; ordered steps; cumulative/remaining math; ROI sign & `None` on free buys; seed reconstruction; trajectory ends at `final_score`; empty-path plan.
- CLI (`tests/test_cli.py`): build-mode table + json; upgrade-mode table + json; `--write` loadable; bad budget clean; infeasible budget clean; mode-conflict clean; real-DB CI-guard (`total_spent <= budget`, `score_delta > 0` per step).
- ≥80% coverage maintained; no `Traceback` leaks; all existing tests stay green.
